## 2. SYS_EXIT / teardown
`/Users/kevinbarnard/src/minixrs/kernel/src/system/do_exit.rs`

**Signature** (lines 72-77), dispatched `mod.rs:200`:
```rust
pub(super) fn do_exit(
    proc_table: &amp;mut [Proc; N_PROC_SLOTS],
    priv_table: &amp;mut [Priv; NR_SYS_PROCS],
    caller_nr: ProcNr,
    msg: &amp;mut Message,
) -&gt; i32
```

**Address-space teardown** is factored into `teardown_addrspace(ttbr0_pa: u64, dead_asid: u8) -&gt; u64` (lines 209-231) — this is the exact reusable sequence exec needs for destroying the *old* AS after swap:
```rust
fn teardown_addrspace(ttbr0_pa: u64, dead_asid: u8) -&gt; u64 {
    if ttbr0_pa == 0 { return 0; }
    let mut freed: u64 = 0;
    let _ = walk_leaves(ttbr0_pa, &amp;mut |_va, pa, _prot| {
        free_frame(Frame::from_addr(pa));
        freed += 1;
        Ok(())
    });
    unsafe { mmu::flush_tlb_asid(dead_asid) };   // must precede destroy? see note
    AddrSpace { ttbr0_pa }.destroy();
    unsafe { asid::free_asid(dead_asid) };
    freed
}
```
Order (important for exec): **walk_leaves → free each leaf → flush_tlb_asid → AddrSpace{ttbr0_pa}.destroy() → asid::free_asid**. The leaf walk MUST precede `destroy` (destroy frees the tables the walk reads). `AddrSpace { ttbr0_pa }` is constructed by struct literal directly (the struct has a single public field). Note the safety comment (lines 222-224): the dead AS is never the active TTBR0 — for exec you must swap `ttbr0_pa`/ASID on the proc *before* tearing the old one down, so the same "not active" invariant holds.

`free_slot` (lines 237-257) also zeroes sig_pending and bumps generation — exec does NOT free the slot (the proc lives on), so this is not reused, but shows the field-reset pattern.

## 1. PM main loop &amp; handlers

```rust

    NOTIFY_MESSAGE if msg.m_source == system =&gt; drain_ksigs(system),

    PM_FORK =&gt; handle_fork(system, vm, sched, &amp;mut msg),
```

**`pm_init`** (127–137): `sef_publish_to_ds(name)`, then `mproc::seed()`, then `privctl_set_user(boot_endpoint(SYSTEM), boot_endpoint(STUB_E_PROC_NR))` to release the frozen stub E. Returns the privctl result as the init rc.

**`handle_getpid`** (144–157): caller identified purely by kernel-stamped `msg.m_source` (no spoofable payload). Slot = `usize::try_from(endpoint_proc(caller_e).get())`. On hit writes ppid into payload `0..4` and replies `m_type = pid`; miss → `ESRCH`.

**`handle_fork`** (169–230) — the canonical multi-step kernel-call driver, the template exec will follow:
1. Resolve `parent_slot` from `endpoint_proc(parent_e)`; `EINVAL` if bad, `EINVAL` if `!mproc::in_use(parent_slot)` (170–176).
2. `mproc::alloc_slot()` → child slot or `EAGAIN` (180–182).
3. `sys_fork(system, parent_e, child_slot as i32)` → `(rc, child_e)`; on failure `mproc::cleanup(child_slot)` + reply (185–189).
4. `vm_fork(vm, parent_e, child_e)`; on failure roll back with `sys_exit(system, child_e)` + `cleanup` (193–198).
5. `sched_start(sched, child_e, CHILD_PRIORITY, CHILD_QUANTUM)`; failure → `sys_exit` + `cleanup` (202–207). `CHILD_PRIORITY = 8`, `CHILD_QUANTUM = 5` (61–62).
6. `privctl_set_user(system, child_e)` to release the freeze; failure → `sched_stop` + `sys_exit` + `cleanup` (216–222).
7. `mproc::set_child(child_slot, parent_slot, child_e)` → child_pid (224).
8. **Reply to both halves of the shared SENDREC** (227–229):
```rust
reply(child_e, msg, 0);          // child half: m_type = 0
reply(parent_e, msg, child_pid); // parent half: m_type = child pid
```
Ordering is deliberate: release before the replies, replies last, so the child can't run until identity/memory/scheduling are set up. `msg` is reused for both sends (the kernel snapshots each send).

**`handle_exit`** (240–271): caller = `msg.m_source`, status = `rd_i32(msg, 0)`. `sched_stop` then `sys_exit`, then `mproc::set_zombie(child_slot, encode_status(status))`. Exiting child gets **no** reply. `encode_status` (299–301) = `(status &amp; 0xff) &lt;&lt; 8` (MINIX `W_EXITCODE`).

**`handle_wait`** (278–295): zombie child → reply pid with status in payload `0..4` + cleanup; else live child → `set_waiting(true)` and SUSPEND (no reply); else `ECHILD`.

Public wrappers: `ipc_send(dest, &amp;mut Message)` (28), `ipc_receive(src, &amp;mut Message)` (35), `ipc_sendrec(src_dest, &amp;mut Message)` (42), `ipc_sendnb` (49), `ipc_notify(dest)` (58). There is **no** higher-level "kernel call" helper — PM builds each message by hand.

**PM's per-call helper pattern** — every helper builds a `Message` with `m_source: 0`, sets `m_type`, writes i32 args via `wr_i32`, `ipc_sendrec`s, and returns `m.m_type` (the kernel-call result). All in main.rs:

- `sys_fork(system, parent_e, child_nr) -&gt; (i32, Endpoint)` (413–426): `m_type = SYS_FORK`; payload `0..4 = parent_e`, `4..8 = child_nr`. On OK, reads child endpoint from reply payload `0..4`. Returns `(m.m_type, rd_i32(&amp;m, 0))`.
- `vm_fork(vm, parent_e, child_e) -&gt; i32` (431–444): `m_type = VM_FORK`; payload `0..4 = parent_e`, `4..8 = child_e`. Sent to the **VM** endpoint (a server request, not a kernel call).
- `sched_start(sched, target_e, priority, quantum) -&gt; i32` (449–463): `m_type = SCHEDULING_START`; payload `0..4 = target_e`, `4..8 = priority`, `8..12 = quantum`. Sent to **SCHED**.
- `sched_stop(sched, target_e) -&gt; i32` (468–480): `m_type = SCHEDULING_STOP`; payload `0..4 = target_e`.
- `sys_exit(system, target_e) -&gt; i32` (364–376): `m_type = SYS_EXIT`; payload `0..4 = target_e`.
- `sys_endksig(system, target_e)` (380–388), `drain_ksigs(system)` (320–358).
- `privctl_set_user(system, target_e) -&gt; i32` (393–406): `m_type = SYS_PRIVCTL`; payload `0..4 = target_e`, `4..8 = PRIVCTL_SET_USER`.

**Reply helper** `reply(target_e, msg, m_type)` (486–490): sets `msg.m_type = m_type`, `msg.m_source = 0` (kernel overwrites on delivery), `ipc_send(target_e, msg)`. Payload must be written before calling. **Payload accessors** `rd_i32`/`wr_i32` (494–502): native-endian i32 at `payload[off..off+4]`.

**Kernel side of the SYS_FORK call** (the analog a real `do_exec` would mirror): `/Users/kevinbarnard/src/minixrs/kernel/src/system/do_fork.rs`, `do_fork(proc_table, caller_nr, msg) -&gt; i32` (72). Payload table documented at lines 36–42. Trace pattern: `[ksys SYS_FORK] parent=… nr=… child_nr=… child_e=… pages=…` gated by `FORK_TRACE_HEAD=6` / `FORK_TRACE_EVERY=100` (64–68, 187–195).
