## 4. SIGSEGV from a page fault → PM

**Classic (no VM server):** the CPU exception is caught in the kernel exception handler and turned straight into a PM-bound signal via `cause_sig`. `/Users/kevinbarnard/src/learn/rminix/kernel/arch/i386/exception.c`: the exception table maps `"Page fault" → SIGSEGV` (`:42`; also GP fault, stack fault → SIGSEGV, `:40-41`). Chain: **CPU page fault → kernel `exception()` → `cause_sig(proc, SIGSEGV)` → `send_sig(PM, SIGKSIG)` → PM `ksig_pending`/`check_sig`/`sig_proc` → `pm_exit` → `sys_exit`/clear_proc.**

**Modern (with VM server):** the kernel forwards the page fault to VM; VM's `handle_pagefault()` decides, and when it cannot satisfy the fault it explicitly issues the SYS_KILL kernel call for SIGSEGV: `/Users/kevinbarnard/src/learn/minix/minix/servers/vm/pagefaults.c:102-103, 113-114, 146-147` — `sys_kill(vmp-&gt;vm_endpoint, SIGSEGV)`. That SYS_KILL (from VM, a system process, not PM) runs `cause_sig` in the kernel, again notifying PM via SIGKSIG. So: **page fault → kernel → VM (`do_pagefaults`/`handle_pagefault`) → `sys_kill(ep, SIGSEGV)` → kernel `cause_sig` → PM.** VM does *not* call PM directly; it goes back through the kernel.

**Answer to the design question**: for a normal **kill-with-default-terminate**, PM does **not** route through `SYS_KILL`. `SYS_KILL` is the *inbound* path (userland `kill(2)` → PM `do_kill`; and other servers/kernel → PM via `cause_sig`). Once PM decides the disposition is "terminate", it calls `pm_exit → sys_exit` (modern `sys_clear`), i.e. **`SYS_EXIT`/`SYS_CLEAR` is what actually tears the process down**. PM only uses `sys_kill` (SYS_KILL) for the *sig2mess* case (delivering a signal-as-notification to a system process), `signal.c:479`.

## 3. `init_boot_image` — how priv/proc slots are assigned at boot

`table.rs::init()` (:327-330) → `init_empty_slots()` (:332-350, gives every slot a valid `(nr, endpoint)` and every priv slot its `id`) → **`init_boot_image()`** (:352-362).

- `IMAGE: [BootEntry; 16]` (:111-258): 5 kernel tasks + 11 boot procs (PM..INIT). Each `BootEntry { nr, name, priv_flags, trap_mask, priority, quantum_ms, runnable }` (:98-106). **PM's entry** (:159-167): `priv_flags = SYS_PROC|PREEMPTIBLE`, `trap_mask = SRV_T`, `priority = SRV_Q`, `runnable = false`.
- **`n_active = N_IMAGE = 16`** (:355) — the count `populate_priv` uses to fill `ipc_to`.
- `populate_priv(id, entry, n_active)` (:364-386): priv slot index == IMAGE index (PM → priv slot 5). For `SRV_T` slots it `fill_bits(ipc_to, 16)` (send to priv slots 0..15) and `fill_bits(k_call_mask, NR_KERN_CALLS_PHASE4=15)` (**every kernel call, incl. SYS_PRIVCTL**). **`sig_mgr = boot_endpoint(RS_PROC_NR)`** for every slot (:383).
- `populate_proc` (:388-412): sets `priv_id`, priority, quantum, endpoint, `scheduler = NONE`, name, and `rts_flags = RTS_NO_PRIV` (boot servers start blocked until an ELF loads).

**Consequence for PM:** its proc slot (0) and priv slot (5, SRV_T, full ipc_to over slots 0-15, full k_call_mask) are already wired by `init_boot_image`. But `ipc_to` covers only priv slots 0..15 — **not** the stub priv slots 16-19. To *signal* a stub (SEF_SIGNAL send) PM would need that reverse edge opened, exactly as `install_stub_d_priv` opens VM→D (see below). A kernel *terminate* of a stub via `SYS_PRIVCTL`/new call needs no ipc_to edge.

### Stub install (`kernel/src/arch/aarch64/userland.rs`)
- Stub proc nrs: `STUB_A=11, B=12, C=13, D=14` (:76-82); priv ids `A=16, B=17, C=18, D=19` (:86-89).
- `userland_bootstrap()` (:132-242): loads MXBI servers via `load_boot_server` (:265-314, clears `RTS_NO_PRIV`, enqueues), `build_stub` ×4, pre-delegates stub C to SCHED (`p.scheduler = boot_endpoint(SCHED_PROC_NR)`, :218-221), then `install_stub_privs()` (:463-480).
- `install_stub_c_priv` (:488-512): `USR_T`, `ipc_to`→SYSTEM only, `k_call_mask`→`SYS_GETINFO` only. `sig_mgr = RS`.
- **`install_stub_d_priv` (:530-563)**: `USR_T`, `ipc_to`→VM only, empty `k_call_mask`; then **opens the reverse VM→D edge** with a separate `priv_slot_mut(vm_priv_id)` borrow and `set_sys_bit(&amp;mut vm_pr.ipc_to, STUB_D_PRIV_ID)` (:559-562). This is the reference pattern for opening a cross-priv IPC edge (e.g. PM→stub for signal delivery). Stub D (priv 19) is the process that takes the VM SIGSEGV path.

## 4. Kernel-call dispatch (`kernel/src/system/mod.rs`)

- `kernel_call_sendrec` (:70-135): the `SENDREC → SYSTEM` fast path. Enforces the same `ipc_to` gate as `mini_send` (:94-99), copies request from user (`copy_msg_from_user`), **stamps `msg.m_source` with the verified caller endpoint** (:109), dispatches, then writes the reply (`m_source = SYSTEM`, `m_type = result`) back to the user buffer, emitting the `[ksys N]` trace (:124-132).
- `kernel_call_dispatch` (:147-182): range-checks `m_type - KERNEL_CALL` against `NR_SYS_CALLS` (→`EBADREQUEST`), gates on `k_call_mask` via `get_call_bit` (→`ECALLDENIED`), then a **target-taking match** routes `SYS_VMCTL / SYS_SCHEDULE / SYS_SCHEDCTL` (:170-175) — these get the whole `&amp;mut [Proc; N_PROC_SLOTS]` + `caller_nr`. Everything else falls to `dispatch_caller_local`.
- `dispatch_caller_local` (:188-210): const-asserts `NR_KERN_CALLS_PHASE4 == 15` to force a new arm per new call. **`SYS_PRIVCTL =&gt; stubs::do_privctl(caller, caller_priv, msg)`** (:195).

**Current `SYS_PRIVCTL` (`kernel/src/system/stubs.rs:25`):** `enosys_stub!(do_privctl)` → returns `ENOSYS` (:17-23). Signature is caller-local: `fn(_caller: &amp;mut Proc, _caller_priv: &amp;Priv, _msg: &amp;mut Message) -&gt; i32`. **To make it real and act on a *target* named in the message (MINIX `SYS_PRIVCTL` targets another proc), it must move to the target-taking route** — add a `SYS_PRIVCTL =&gt; do_privctl::do_privctl(proc_table, caller_nr, msg)` arm in `kernel_call_dispatch` alongside VMCTL/SCHEDULE (:170-175), taking `&amp;mut [Proc; N_PROC_SLOTS]` (and likely a `&amp;mut [Priv]` — note the current signature passes only `&amp;Priv`; the dispatch call site at :179-181 borrows `priv_table` immutably, so wiring PRIVCTL to mutate priv slots needs the mutable priv slice like `kernel_call_sendrec` holds at :71).

### Pattern references

**`do_schedule.rs`** (target-taking): `do_schedule` (:71-131) and `do_schedctl` (:140-171). Reads target endpoint (`SELF`→caller) at payload `0..4` via `read_i32`, resolves `endpoint_proc → proc_index` (→`EINVAL` if OOB), mutates the slot, then uses the **capture-then-borrow-end** `sched::rts_unset`/`dequeue`/`enqueue` pattern (:104-119). Head-carve trace `[ksys SYS_SCHEDULE]` with `SCHED_TRACE_HEAD=6` (:52-58, :121-129). `do_schedctl` snapshots the caller endpoint before mutably borrowing the target (:159-162) to handle self-targeting.

**`do_setalarm.rs`** (caller-local): `do_setalarm(caller, _priv, msg)` (:38-59) reads `delta` (u64) at payload `0..8`, computes `at = now + delta`, sets `caller.alarm_at`, calls `clock::arm_alarm(at)`, replies with previous remaining ticks at payload `0..8`. `read_u64`/`write_u64` helpers (:61-73). This is the template for a caller-local getpid-style handler if any PM logic were kernel-side (it won't be — getpid lives in PM's mproc table).

**`do_vmctl.rs`** documents the trust model (:17-28): the only gate is `k_call_mask`; a caller reaching the handler may name any target. This is the model `SYS_PRIVCTL` will inherit (PM is the sole intended holder).

## 5. Termination mechanics that exist today

**There is no kill/terminate/exit path.** Nothing clears a proc slot, frees a priv slot, or bumps a generation. The building blocks that exist:

- **`sched::rts_set(p, flag)`** (`kernel/src/proc/sched.rs:200-208`): `fetch_or`s the flag; if the proc was runnable it `dequeue`s it. Converse `rts_unset` (:220-228) enqueues when all bits clear. These are the capture-`nr`-then-re-borrow helpers (CLAUDE.md:128).
- `dequeue(nr)` (:137-184) splices a proc out of its run-queue band; `pick_proc` (:235-239). `schedule_next` (:292-320) installs the next proc's TTBR0/ASID and asserts `ttbr0_pa != 0 &amp;&amp; asid != 0`.
- **A minimal "terminate"** (default action of the 4.5 kill path) needs, at minimum: `rts_set` a permanent block flag (candidates: `RTS_NO_ENDPOINT=0x100`, `RTS_PROC_STOP=0x2`, or a new terminate flag) so `dequeue` pulls it off the run queue and it never runs again. That already gives "the faulter stops." Fuller teardown (reclaim slot, generation bump for reuse) is slice 4.6 territory.
- `Proc::EMPTY` (`proc_struct.rs:127-162`, `rts_flags=RTS_SLOT_FREE`) and `populate_proc`/`populate_stub_slot` show what a slot holds; there is **no `clear_slot`/free helper yet**.
- **`AddrSpace::destroy` DOES exist** (`kernel/src/arch/aarch64/addrspace.rs:149-155`: `free_subtree(ttbr0_pa,0)` + `free_frame(L0)`), but it is **never called at runtime** — every boot server and stub `mem::forget`s its `AddrSpace` (userland.rs:310, :384) precisely so `destroy` won't run. The plan reserves `AddrSpace::destroy` for `SYS_EXIT` teardown in **slice 4.6** (docs/plan.md:963-965). So a 4.5 terminate can leave the AS mapped (just block the proc); AS teardown is explicitly a later slice.

**The consequence 4.5 gives real:** VM's SIGSEGV path (`servers/vm/src/main.rs:130-135`): an out-of-region fault returns without `VMCTL_PT_MAP`/`VMCTL_CLEAR_PAGEFAULT`, leaving the faulter blocked on `RTS_PAGEFAULT` forever. Region check is `region::contains(nr, far)` (:132). Comment at :126-129 literally names the gap: "the only 'kill' available until PM + signals in Phase 4." PM terminating the faulter is the real consequence.

## 6. Page-fault path and kernel-originated delivery

- **`do_page_fault`** (`kernel/src/arch/aarch64/exception.rs:148-216`): decodes EC (0x20 instr / 0x24 data abort — else halt via `el0_sync_unexpected`), builds `flags` (`PFF_INSTR/PFF_WRITE/PFF_PERMISSION`), **permission faults halt** (:176-178). Under a tightly-scoped borrow it records `p.page_fault_state = {addr:far, flags, ip:elr}` and `sched::rts_set(p, RTS_PAGEFAULT)` (:186-200), captures `faulting_nr`/`name`, then calls `crate::ipc::send_pagefault_to_vm(faulting_nr, far, flags)` (:208) and prints **`[pf] proc=… far=… flags=… -&gt; VM`** every fault (:210-215).
- `send_pagefault_to_vm` (`kernel/src/ipc/mod.rs:289-295`) materializes the proc table and calls **`send::mini_pf_send`** (`kernel/src/ipc/send.rs:139-190`): builds `Message { m_source = faulter endpoint, m_type = VM_PAGEFAULT, payload: far@0..8 (u64), flags@8..12 (u32) }`. **No `ipc_to`/permission check and no deadlock check** — kernel-originated (:134-138). If VM `will_receive`, deliver immediately (`deliver_msg`, `MF_DELIVERMSG`, `rts_unset RTS_RECEIVING`); else queue the faulter on VM's `caller_q` with `rts_set RTS_SENDING` (RTS_PAGEFAULT already keeps it blocked).
- **What VM knows about the faulter:** only `m_source` (the faulter's endpoint). VM does `endpoint_proc(faulting_e).get()` → proc nr (`vm/main.rs:130-131`) and gates on its region table. It can also pull fault addr/flags/ip via `SYS_VMCTL(VMCTL_GET_PAGEFAULT)` (`do_vmctl.rs:218-232`).
- **Sibling kernel-originated patterns** (all bypass `ipc_to`, same shape PM's signal delivery to a `sig_mgr`/target could mimic):
  - `notify::deliver_alarm` (`kernel/src/ipc/notify.rs:119-155`): kernel `NOTIFY` from `CLOCK`, no permission check; immediate-deliver-or-set-`notify_pending`.
  - `send_no_quantum` (`ipc/mod.rs:305-311`) → `mini_sched_no_quantum_send` (`send.rs:212-264`): kernel SEND of `SCHEDULING_NO_QUANTUM` from the preempted proc to its `scheduler`, `m_source` = preempted proc.
  - `will_receive(dst, src_e)` (`notify.rs:29-36`) and `build_notify_message` (:41-47) are the shared delivery predicates.

## 7. Tracing conventions

All traces use an `AtomicU64` counter + a **head carve-out** (first N unconditionally) and/or **modulo sampling**; head carve-out exists because stub C's `SYS_GETINFO` flood would otherwise starve the modulo sampler of rare events.

- **`[ipc N]`** — `ipc/mod.rs::do_ipc` (:113-120). `TRACE_HEAD=12` (:70) + `TRACE_EVERY=100` (:62): `if n &lt;= TRACE_HEAD || n % TRACE_EVERY == 0`.
- **`[ksys N]`** — `system/mod.rs::kernel_call_sendrec` (:124-132). `KCALL_TRACE_EVERY=100` (:57), modulo only. Plus per-handler head-carve variants: `[ksys SYS_SCHEDULE]` (`do_schedule.rs:121-129`, `SCHED_TRACE_HEAD=6`), `[ksys VMCTL_PT_MAP]`/`[ksys VMCTL_PT_UNMAP]` (`do_vmctl.rs:157-201`, `VMCTL_TRACE_HEAD=6`).
- **`[alarm N]`** — `ipc/mod.rs::trace_alarm_fire` (:375-387). `ALARM_TRACE_HEAD=6` + `ALARM_TRACE_EVERY=100` (:315-320); prints only on head or modulo (`owner=… nr=… at=…`).
- **`[noq N]`** — `proc/sched.rs::reschedule` (:392-403). `NOQ_TRACE_HEAD=6` (:39), head-only (delegated quantum-exhaust events are rare).
- **`[pf]`** — `exception.rs::do_page_fault` (:210-215). **Unsampled** — printed on every fault.
- **`[as]`** — boot address-space summary: `userland.rs::load_boot_server` (:298-306) and `print_addrspace_summary` (:604-611).

New 4.5 traces (PM lives at EL0 and cannot print) should follow the same kernel-side counter+head/modulo idiom; a `SYS_PRIVCTL` head-carve trace would mirror `do_schedule.rs`'s `SCHED_TRACE_HEAD` block.

## (e) Risks / gotchas

1. **Borrow discipline in `cause_sig` (two procs touched)**: target mutation (`sig_pending` + `rts_set`) and PM delivery (`&amp;mut proc_table[pm_idx]` + `&amp;mut priv_table[..]`) must be strictly sequential scoped borrows on the *passed-in slices*. Never call `proc_slot_mut`/`proc_table_mut_slice` inside `do_sig`/`deliver_ksig` — the dispatch already holds the exclusive slices; re-materializing is the two-`&amp;mut`-from-one-`UnsafeCell` UB CLAUDE.md line 126 exists to prevent. `deliver_alarm` (notify.rs:119-155) is the exact template. `rts_set` on the already-`RTS_PAGEFAULT`-blocked D is safe (`was_runnable == false`, no dequeue).
2. **`kernel_call_dispatch` priv param must go `&amp;mut`** (not in the brief): do_kill's deferred branch writes PM's `notify_pending`. Mechanical — mod.rs:115 already holds `&amp;mut`; the k_call_mask read and `dispatch_caller_local`'s `&amp;Priv` reborrow are unaffected.
3. **SYSTEM's priv slot exists** — PrivId 3 (IMAGE order asyncm/idle/clock/system/hardware = priv 0-4), populated by `proc::init` with `CSK_T` and empty `ipc_to`. The deferred `notify_pending` path and `take_pending_notification`'s priv→proc→endpoint resolution work unmodified; that empty `ipc_to` is precisely why `deliver_ksig` must bypass `mini_notify` (same reasoning as CLOCK/`deliver_alarm`).
4. **GETKSIG/ENDKSIG livelock hazard**: `do_getksig` clears `sig_pending` but leaves RTS flags, so its scan **must** require `sig_pending != 0` (not just the RTS bit), and PM **must** ENDKSIG every endpoint GETKSIG returns — including `SkipPrivileged`/`NotFound` targets — or the same proc blocks the drain (or, worse, an un-ENDKSIG'd skipped proc keeps `RTS_SIG_PENDING` forever). Conversely ENDKSIG on a *runnable* target would enqueue it via `rts_unset`; in 4.5 every killed target also carries `RTS_PAGEFAULT` and (post-EXIT) `RTS_PROC_STOP`, so it stays off the queue — keep the EXIT-before-ENDKSIG order in PM.
5. **Frozen E vs `schedule_next`'s ttbr0 assert** (sched.rs:310-317 + the hard assert in `switch_ttbr0_with_asid`): build E's AddrSpace at boot like every stub — freeze only via `RTS_NO_PRIV` + skipped enqueue + `priv_id = None`. Do not defer AS construction to PRIVCTL time. E is unreachable by `pick_proc` until `rts_unset` enqueues it, by which time ttbr0/asid are long set.
6. **PM SENDRECs from `init_fresh`**: fine (each SENDREC-to-SYSTEM is synchronous in the caller's EL1 context). Side effect: PRIVCTL enqueues E immediately, and E (band 8) may SENDREC PM before PM reaches `sef.receive` — E simply parks on PM's `caller_q` (E's `ipc_to` is only PM; no cycle, no deadlock).
